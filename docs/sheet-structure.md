# LIRA 用スプレッドシート構成（提案）

実データのシート名・列が異なる場合は、`.env` の `SHEET_*` を実タブ名に合わせるか、タブ名に「月次」「入金」「支払」などのキーワードを含めると **LIRA がタブ一覧から自動で推測**します（完全一致を最優先）。列ヘッダーは本ドキュメントの推奨名に揃えるか、列名を LIRA 前提に調整してください。

## 共有設定

1. Google Cloud でサービスアカウントを作成し、JSON キーをダウンロードする。
2. スプレッドシートを「共有」し、サービスアカウントのメール（`xxx@PROJECT.iam.gserviceaccount.com`）に**編集者**以上で招待する。

---

## シート 1: `月次サマリー`（環境変数 `SHEET_SUMMARY`）

月次の集計値を **1 行＝1 ヶ月** で保持します。LIRA は `対象月` が `GET /summary?month=YYYY-MM` と一致する行を読みます。

| 列（ヘッダー行） | 例 | 説明 |
|------------------|-----|------|
| 対象月 | `2026-05` | `YYYY-MM` 形式推奨（`2026/5` 等もパーサで一部対応） |
| 売上合計 | `1200000` | JPY。カンマ・`¥` 可 |
| 経費合計 | `450000` | 同上 |
| 利益 | `750000` | 未入力なら 売上−経費 で算出可（MVP はシート値優先） |
| 利益率 | `0.625` または `62.5%` | 未入力なら 利益÷売上 で補完可 |

**推奨範囲例**: `A1:F100`（1 行目ヘッダー、2 行目以降データ）

既存の「PL」「月次」シートがある場合: 上記列を追加するか、別シート `月次サマリー` を LIRA 専用に切り出し、`IMPORTRANGE` / `QUERY` で既存シートから同期すると運用が楽です。

---

## シート 2: `入金予定`（環境変数 `SHEET_RECEIVABLES`）

売掛・入金スケジュール。**1 行＝1 件**。

| 列（ヘッダー） | 説明 |
|----------------|------|
| クライアント | 任意。通知文面で使用 |
| 案件名 / 請求書番号 | 任意。識別用 |
| 請求金額 | JPY。未入金一覧・リマインドに使用 |
| 入金予定日 | `YYYY-MM-DD` 推奨 |
| 入金日 | 実績日。未入金なら空 |
| 入金ステータス | 例: `未入金` / `入金済` / `一部入金`（未払い判定に使用） |
| 入金確認チェック | 例: `済` / `未`。内部チェック用 |
| 未入金フラグ | `TRUE` / `FALSE` または `1` / `0`。`GET /unpaid` の明示フラグ |
| メモ | 自由記述 |
| 最終通知日 | 督促送付日。督促文に反映 |

**未入金の判定（MVP）**: 次のいずれかを満たす行を未入金候補とする。

- `未入金フラグ` が真
- または `入金ステータス` が `入金済` 以外で、`入金日` が空

**入金予定リスト**（`/receivables`）: `入金予定日` が今日以降（または当月）など、API クエリで絞り込み可能な設計。

---

## シート 3: `支払予定`（環境変数 `SHEET_PAYABLES`）

買掛・支払スケジュール。**1 行＝1 件**。

| 列（ヘッダー） | 説明 |
|----------------|------|
| 支払予定日 | `YYYY-MM-DD` |
| 支払先 | 取引先名 |
| 支払金額 | JPY |
| 支払ステータス | 例: `未払` / `支払済` |
| メモ | 自由記述 |

**支払予定リスト**: `支払ステータス` が未完了の行を中心に、`支払予定日` でソートして返却。

---

## 典型会計シートとの差分

| 典型 | LIRA での扱い |
|------|----------------|
| 総勘定元帳の羅列 | そのままでは読みにくい → **集計シート** `月次サマリー` を用意 |
| 請求と入金が同一シート | 列名を上記に合わせるか、LIRA 専用ビューシートを複製 |
| 経費が複数タブ | `月次サマリー` の `経費合計` に既に集約された値を載せる（MVP はシート側集計前提） |

---

## セル参照ではなく「行ベーステーブル」にする理由

- サービスアカウント経由の Sheets API は **範囲一括取得** が簡潔で、列追加にも強い。
- 列名をヘッダー行に置くと、運用担当が列順を多少変えてもマッピング調整がしやすい。

## サンプル行（イメージ）

**月次サマリー**

```text
対象月,売上合計,経費合計,利益,利益率
2026-05,1200000,450000,750000,62.5%
```

**入金予定**

```text
クライアント,案件名,請求金額,入金予定日,入金日,入金ステータス,入金確認チェック,未入金フラグ,メモ,最終通知日
株式会社A,Web制作,330000,2026-05-20,,未入金,未,TRUE,,2026-05-10
```

**支払予定**

```text
支払予定日,支払先,支払金額,支払ステータス,メモ
2026-05-25,外注B株式会社,88000,未払,5月分
```
